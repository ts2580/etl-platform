상세 조회

좌측에서 저장소를 클릭하면 상세가 보여져요.

복호화 API 예시

POST /api/storages/databases/{storageId}/decrypt
{
  "userKey": "your-password",
  "revealRaw": false
}

응답은 기본 마스킹 상태입니다. 평문이 필요하면 토큰 기반 1회 조회를 사용해요.

저장된 자격 증명 조회

복호화는 서버의 APP_DB_CREDENTIAL_KEY 설정을 사용해요. 별도 사용자 키 입력 없이 조회할 수 있습니다.

복호화 키를 입력하고 조회해 주세요.
